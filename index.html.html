How Image LSB Steganography Works
01
What is LSB?
Every pixel has RGB values (0–255). The Least Significant Bit is the final bit of each value. Flipping it changes color by only ±1 — completely invisible to human eyes.
R: 11001010 → 11001011 (bit hidden)
G: 10110100 → 10110100 (bit hidden)
B: 01110011 → 01110011 (bit hidden)
02
Encoding Steps
The message converts to binary. Each bit replaces one LSB in a pixel channel. A null-byte end marker signals extraction stop. Must export as PNG (lossless).
1. Message → ASCII → Binary bits
2. Each bit replaces 1 LSB in pixel
3. Append 8× null-byte end marker
4. Export as lossless PNG

How Text Unicode Steganography Works
01
Zero-Width Characters
Unicode includes invisible characters — Zero Width Space (U+200B) and Zero Width Non-Joiner (U+200C). They are invisible in any editor or chat app but encode binary data.
bit 0 → U+200B (Zero Width Space)
bit 1 → U+200C (Zero Width Non-Joiner)
end → U+200D (Zero Width Joiner)
02
Encoding Steps
The secret message converts to binary. Each bit becomes an invisible Unicode character inserted between words of the cover text. Output looks exactly like normal text.
1. Secret message → binary bits
2. Bits → invisible Unicode chars
3. Insert between cover text words
4. Output looks 100% normal

Extension Detection

How File Corruption Works
01
DOCX / PPTX are ZIP files
Office Open XML formats (.docx, .pptx, .xlsx) are actually ZIP archives containing XML files. So renaming .docx → .pptx doesn't just confuse the extension — it gives PowerPoint the wrong XML structure to parse.
docx → ZIP with word/ folder inside
pptx → ZIP with ppt/ folder inside
mislabeled → App reads wrong XML, fails
02
Magic Bytes & Headers
Wiping the header removes the file's identity. The OS and apps check the first few bytes to decide how to handle a file. Zero those out and even renaming it back won't help — the signature is gone.
1. Original magic bytes intact
2. Overwrite first 8 bytes → 00 00...
3. App sees unknown format → error
4. Detector shows "Unknown Binary"

How Hashing Works — QuickHash-GUI Algorithms
MD5
Message Digest 5
128-bit output. Fast but cryptographically broken — used for checksums, not security. QuickHash uses it for legacy file verification.
SHA1
Secure Hash Algorithm 1
160-bit output. Deprecated for security use since 2017. Still common in Git and legacy systems. Collision attacks are known.
SHA256
SHA-2 Family · 256-bit
The current gold standard for file integrity. Used by Bitcoin, TLS, code signing. QuickHash uses this as its primary verification algorithm.
SHA512
SHA-2 Family · 512-bit
Stronger variant of SHA-256. 512-bit output. Preferred for high-security applications and large file verification.
SHA-3
Keccak Algorithm
Completely different design from SHA-2. Chosen by NIST in 2015. Resistant to length-extension attacks. QuickHash includes 256-bit variant.
CRC32
Cyclic Redundancy Check
32-bit checksum — not cryptographic but extremely fast. Used in ZIP files, network packets, and storage verification. QuickHash includes it for quick integrity checks.
BLAKE2b
Modern Cryptographic Hash
Faster than MD5 yet more secure than SHA-3. Used in password hashing (Argon2), WireGuard, and file systems. QuickHash v3+ includes Blake2B and Blake3.
Base64
Binary-to-Text Encoding
Not a hash — an encoding. Converts binary data to ASCII-safe text. Used in email attachments, JWT tokens, and data URIs. QuickHash includes Base64 decode since v2.8.3.

How Password Analysis Works
01
Entropy Score
Entropy measures unpredictability in bits. Every character from a larger charset exponentially increases the search space. 70+ bits is considered very strong against modern attacks.
26 lower: log2(26) ≈ 4.7 bits/char
+digits: charset → 36 ≈ 5.2 bits/char
+symbols: charset → 95 ≈ 6.6 bits/char
02
Crack Time Estimate
Based on GPU-scale brute-force rates (~10 billion hashes/sec for MD5). Dictionary and pattern attacks can shortcut this dramatically — which is why entropy alone isn't enough.
40-bit: Seconds to minutes
60-bit: Years to millennia
80+ bit: Cosmological timescales

Encoding Formats Explained
B64
Base64
3 bytes → 4 ASCII chars. ~33% size overhead. Foundation of HTTP Basic Auth, JWT, email attachments, and inline data URIs in HTML.
B32
Base32
5 bits per char, case-insensitive, no ambiguous chars. Used in TOTP seed keys (Google Authenticator), .onion addresses, and DNS labels.
HEX
Hexadecimal
Each byte → 2 hex digits (00–FF). Common in debuggers, color codes, cryptographic hash output, network protocols, and memory dumps.
URL
URL Encoding
Replaces reserved chars with %XX sequences. Ensures safe transmission of query strings, form data, and Unicode characters in HTTP.
B58
Base58
Bitcoin's encoding format. Removes confusing chars (0 vs O, I vs l). Used in wallet addresses, IPFS content IDs, and Flickr photo IDs.
B85
ASCII85 / Base85
4 bytes → 5 chars. More efficient than Base64 (~25% overhead). Used inside PDF and PostScript for embedding binary data in text streams.
📖 Code Architecture & Developer Reference
VaultBlack — How the Code Works, Tool by Tool
This section explains the actual algorithms and browser APIs powering each tool. All processing runs locally — no server, no uploads, no tracking. Everything is pure client-side JavaScript using Web Crypto API, the Canvas API, and pure-JS implementations of cryptographic algorithms.
T1
Image LSB Steganography
Uses the HTML5 Canvas API — getImageData() reads raw pixel bytes into a Uint8ClampedArray (R,G,B,A per pixel). Encoding replaces the last bit of each R/G/B channel value with one bit of the secret message binary. Each pixel can hide 3 bits. A 64-zero-bit null terminator marks the message end. The output is exported as lossless PNG via canvas.toDataURL('image/png') — JPEG would destroy the hidden bits through lossy compression. Decoding reads all LSBs back into a bitstring and converts 8-bit groups to ASCII characters.
T2
Text Unicode Steganography
Converts the secret message to binary. Each bit is mapped to an invisible Unicode character: U+200B (Zero Width Space) = bit 0, U+200C (Zero Width Non-Joiner) = bit 1. These are distributed between words of the visible cover text. A U+200D Zero Width Joiner marks the end of the hidden data. These characters are rendered as zero-width glyphs — invisible in every browser, editor, and messaging app, yet fully preserved when copied and pasted.
T3
File Scanner Pipeline
Step 1 reads the first 512 bytes as Uint8Array via FileReader.readAsArrayBuffer() and matches against 50+ magic byte signatures (e.g. 89 50 4E 47 = PNG). Step 2 uses the browser-native crypto.subtle.digest() Web Crypto API for SHA-1/256/512 (hardware accelerated), plus a pure-JS RFC 1321 MD5 implementation (browsers don't expose MD5 in SubtleCrypto). Step 3 is a heuristic risk engine that scores filenames against suspicious keyword lists, checks extension mismatches, and auto-builds VirusTotal/MalwareBazaar URLs with the SHA-256 pre-filled.
T4
File Corruptor
Reads the entire file into a Uint8Array via FileReader.readAsArrayBuffer(). Four corruption modes: Extension Only — content unchanged, only the download attribute filename changes. Header Wipe — overwrites the first 8 bytes with 0x00, destroying the magic signature. Partial Corrupt — randomly selects 5% of byte indices and replaces with Math.random()*256 values. Full Randomize — fills a fresh Uint8Array via crypto.getRandomValues(). All modes create a Blob object-URL for browser download.
T5
Hash Generator
Eight algorithms: SHA-1/256/512 use crypto.subtle.digest() (browser-native, hardware accelerated). MD5 is a complete pure-JS RFC 1321 implementation (using 32-bit integer arithmetic). CRC-32 pre-computes a 256-entry lookup table via bit operations and uses it for streaming. BLAKE2b-256 implements the full reference spec with BigInt64Array for 64-bit lane operations. SHA-3 uses the Keccak-f[1600] permutation with 24 rounds. Base64 uses browser-native btoa()/atob(). Live hashing debounces on a 300ms timer using setTimeout().
T6
Password Analyzer
Calculates the character set size based on which character classes are present (lowercase=26, uppercase=26, digits=10, symbols=33). Entropy is computed as log2(charset) × length bits. Crack time is estimated by computing total combinations (charset^length) and dividing by 10 billion (realistic GPU brute-force rate for MD5). Breach detection compares against a hardcoded list of the top 30 most commonly breached passwords, plus regex patterns for keyboard walks (qwerty, asdfg), sequential runs, embedded years, and common password keywords. A weighted score 0–100 combines length, charset diversity, and pattern penalty deductions.
Reads up to 64 KB via FileReader.readAsArrayBuffer(). For JPEG files, scans for the 0xFF 0xE1 APP1 marker (EXIF block start). Detects byte order by checking bytes 10–11 of the EXIF block (0x4949 = Intel LE, 0x4D4D = Motorola BE). Walks the IFD (Image File Directory) entries — each 12 bytes containing a tag ID, data type, count, and value/offset. Flags privacy-sensitive tags: GPS IFD (0x8825), Camera Make (0x010F). For PDFs, reads the first 4 KB as Latin-1 text and applies regex patterns to match XMP/Info dictionary entries like /Author (name) and /CreationDate (D:20240101).
T7
Base Encoder / Decoder
Base64 uses browser-native btoa()/atob(). Base32 manually processes 5-bit groups from input bytes, mapping to RFC 4648 alphabet (A–Z, 2–7) with = padding. Base58 implements Bitcoin's positional base-conversion: repeatedly divides the byte value by 58 accumulating remainders, using the unambiguous B58_CHARS alphabet (no 0, O, I, l). ASCII85 maps every 4 input bytes to 5 output chars in range ASCII 33–117, with a special 'z' shorthand for all-zero groups. URL encoding uses the standard encodeURIComponent()/decodeURIComponent(). Hex encodes each byte to 2 lowercase hex digits via .toString(16).padStart(2,'0').
UI
Theme System & Tab Router
The tab system uses CSS class toggling: switchType() removes active from all .tab-c divs then adds it to the target. The Monte Carlo dark theme works via a :root.monte-carlo CSS selector block that overrides all custom properties (--ink, --paper, --accent, etc.) set on the base :root. Toggling document.documentElement.classList switches the entire theme without a page reload. Drag-and-drop uses dragover + drop event listeners with e.dataTransfer.files and DataTransfer injection to set file inputs programmatically.
